# 收藏功能设计

## 概述

为剪切板助手新增"收藏"功能，用户可将文本历史记录收藏到自定义分组中，收藏永久保存，不受历史记录上限影响。分组支持新建、重命名、删除（级联删除收藏项）。

## 需求

- 新增"收藏"Tab，展示分组及分组下的收藏内容
- 从文本历史记录中可收藏到指定分组
- 收藏记录永久保存，无数量上限
- 分组可管理：新建、重命名、删除
- 删除分组时级联删除其下所有收藏项
- 仅支持文本收藏，不支持图片收藏
- 适配剪切板窄窗口布局

## 数据层

### 已有表（无需迁移）

```sql
Groups (id UUID PK, name VARCHAR(255), createTime DATETIME)
GroupItems (id UUID PK, groupId VARCHAR(255), content TEXT,
            title VARCHAR(255), createTime DATETIME, updateTime DATETIME)
```

### db_service.js 新增方法

| 方法 | 功能 |
|------|------|
| `addGroup(name)` | 创建分组 |
| `fetchGroups()` | 查询所有分组（按创建时间排序） |
| `renameGroup(id, name)` | 重命名分组 |
| `removeGroup(id)` | 删除分组 + 级联删除该分组下所有 GroupItems |
| `addGroupItem(groupId, content, title)` | 添加收藏项 |
| `fetchGroupItems(groupId)` | 查询分组下的收藏项（按创建时间倒序） |
| `removeGroupItem(id)` | 删除单条收藏项 |

收藏项无数量上限，永久保存。

## UI 结构

### 收藏 Tab（FavoritePage.vue）

在 CutPage.vue 的 Tabs 中新增"收藏"Tab，加载 FavoritePage.vue 组件。

**布局：分组下拉切换 + 单列列表**

```
┌──────────────────────────────┐
│ [文本] [图片] [收藏] [白板]   │
├──────────────────────────────┤
│  [常用代码 ▼]  [+新建] [⚙管理] │  ← 分组下拉 + 操作
├──────────────────────────────┤
│  1. const a = ...    3分前    │
│  2. npm install...   1小时前  │
│  3. https://...      昨天     │
│  4. ...                      │
└──────────────────────────────┘
```

**顶部操作栏：**
- 分组下拉选择器（a-select）：切换当前查看的分组
- 「+新建」按钮：弹出输入框创建新分组
- 「⚙管理」按钮：弹出管理弹窗

**内容列表：**
- 单列虚拟滚动列表（vue-virt-list），与文本 Tab 风格一致
- 每条：文本内容截断 + 相对时间 + 操作按钮（复制/删除）
- 无分组选中或分组为空时显示空状态提示

### 分组管理弹窗

```
┌── 分组管理 ──────────────┐
│  常用代码    [✏️重命名] [🗑️] │
│  临时笔记    [✏️重命名] [🗑️] │
│  账号信息    [✏️重命名] [🗑️] │
│                           │
│            [关闭]          │
└───────────────────────────┘
```

每行分组名 + 重命名按钮 + 删除按钮。重命名点击后行内编辑（a-input 替换文字）。

### 从文本历史收藏到分组

TimeList.vue 中现有「分组」菜单项，打通完整流程：

1. 文本列表每条 `⋯` 菜单 → 点击「分组」
2. 弹出分组选择弹窗（a-modal）：
   - Radio 单选列表展示所有分组
   - 底部「+ 新建分组」快捷输入框
   - 选中分组后点「确认」
3. 调用 `db_service.addGroupItem()` 保存收藏
4. 成功提示「已收藏到 xxx 分组」

**收藏行为：**
- 收藏内容为副本（content 字段），非引用。原文本历史记录删除不影响收藏
- 同一文本可收藏到多个分组
- title 取内容前 50 字符截断
